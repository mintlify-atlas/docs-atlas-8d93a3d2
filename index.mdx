## What is Semgrep?

Semgrep is a fast, open-source static analysis tool that searches code, finds bugs, and enforces secure guardrails and coding standards. It supports 30+ languages and runs in an IDE, as a pre-commit check, and as part of CI/CD workflows.

Semgrep is **semantic grep for code**. While running `grep "2"` would only match the exact string `2`, Semgrep matches `x = 1; y = x + 1` when searching for `2`. Rules look like the code you already write — no abstract syntax trees, regex wrestling, or painful DSLs.

```yaml Example Semgrep rule
rules:
  - id: print-to-logger
    patterns:
      - pattern: print(...)
    message: |
      Found a print() statement. Consider using a logger instead
      for better log management and control.
    languages: [python]
    severity: WARNING
```

## Get started

<Steps>
  <Step title="Install Semgrep">
    Install using your preferred method:
    ```bash
    # macOS
    brew install semgrep

    # pip (Linux/macOS/Windows)
    python3 -m pip install semgrep

    # Docker
    docker pull semgrep/semgrep:latest
    ```
  </Step>
  <Step title="Run your first scan">
    Scan your project using a community rule pack:
    ```bash
    semgrep scan --config=auto .
    ```
  </Step>
  <Step title="Write a custom rule">
    Write a rule that matches patterns in your own codebase and add it to your workflow.
    See [Writing Rules](/writing-rules/overview) to learn how.
  </Step>
</Steps>

<Note>
  Semgrep analyzes code locally on your computer or in your build environment. **By default, code is never uploaded.**
</Note>

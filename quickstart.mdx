---
title: "Quickstart"
description: "Get Semgrep scanning your code in under 5 minutes."
---

<Steps>
  <Step title="Install Semgrep">
    Install Semgrep using your preferred method. For most users, pip is the easiest option:

    ```bash
    python3 -m pip install semgrep
    ```

    On macOS, you can use Homebrew:

    ```bash
    brew install semgrep
    ```

    Verify the installation:

    ```bash
    semgrep --version
    ```

    See [Installation](/installation) for all methods including Docker and conda.
  </Step>

  <Step title="Run your first scan">
    Navigate to your project directory and run a scan using `--config=auto`. This automatically selects relevant rules from the Semgrep Registry based on the languages in your codebase:

    ```bash
    semgrep scan --config=auto .
    ```

    Semgrep will scan your code and print a summary of any findings. A typical output looks like this:

    ```
    ┌─────────────────────────────────────────────────────────────────────┐
    │ Semgrep findings                                                    │
    └─────────────────────────────────────────────────────────────────────┘

    src/auth.py
       45┆ if password == password:
            ╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌
            python.lang.correctness.useless-eqeq.useless-eqeq
            This expression is always True. Did you mean to compare
            against a different variable?

    ────────────────────────────────────────────────────────────────────────
    Ran 42 rules on 18 files: 1 finding.
    ```

    Each finding shows:
    - The file and line number
    - The matching code
    - The rule ID that triggered
    - A description of the issue
  </Step>

  <Step title="Run a one-off pattern search">
    You can run ad-hoc searches without writing a rule file using the `-e` flag. For example, find all Python comparisons where both sides are identical (a common bug):

    ```bash
    semgrep -e '$X == $X' --lang=py .
    ```

    The `$X` is a **metavariable** that matches any expression. Because the same metavariable appears on both sides of `==`, Semgrep only matches when the two sides are identical.

    Try other quick searches:

    ```bash
    # Find all calls to exec() in Python
    semgrep -e 'exec(...)' --lang=py .

    # Find hardcoded "TODO" strings in JavaScript
    semgrep -e '"TODO"' --lang=js .
    ```
  </Step>

  <Step title="Scan with a rule pack">
    The Semgrep Registry provides curated rule packs for specific languages and frameworks. Run a pack for Python:

    ```bash
    semgrep scan --config=p/python .
    ```

    Other useful packs:

    ```bash
    # Security-focused rules
    semgrep scan --config=p/security-audit .

    # OWASP Top 10
    semgrep scan --config=p/owasp-top-ten .

    # JavaScript and TypeScript
    semgrep scan --config=p/javascript .
    ```

    Browse all available packs at [semgrep.dev/explore](https://semgrep.dev/explore).
  </Step>

  <Step title="Write your first rule">
    Create a file called `my-rules.yaml` with the following rule that finds Python `print()` statements:

    ```yaml my-rules.yaml
    rules:
      - id: no-print-statements
        pattern: print(...)
        message: |
          Found a print() statement. Use a logger instead for
          better log management and control.
        languages: [python]
        severity: WARNING
    ```

    Run it against your code:

    ```bash
    semgrep scan --config=my-rules.yaml .
    ```

    Rules use the same pattern syntax as the `-e` flag, but stored in YAML files so you can version and share them. Metavariables, ellipsis operators, and pattern combinators give you fine-grained control over what matches.
  </Step>
</Steps>

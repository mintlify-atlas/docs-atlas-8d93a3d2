## Generic mode

Generic mode enables Semgrep to search files that are not supported by a dedicated parser. It uses whitespace-aware string matching (via the `spacegrep` engine) rather than AST-based analysis.

Use `generic` in the `languages:` field to target any file type:

```yaml
rules:
  - id: find-todo-comments
    pattern: "# TODO: $TASK"
    message: "Unresolved TODO: $TASK"
    languages: [generic]
    severity: INFO
```

Generic mode also applies automatically to file types such as ERB templates, Jinja templates, and any file extension not claimed by a dedicated parser.

### Choosing a generic engine

Two generic engines are available via the `options:` field:

| Engine | Key | Description |
|---|---|---|
| spacegrep | `spacegrep` (default) | Whitespace-sensitive structural matching |
| aliengrep | `aliengrep` | Regex-based search over raw text |

```yaml
rules:
  - id: custom-generic-rule
    pattern: "SECRET_KEY = \"...\""
    message: Possible hardcoded secret.
    languages: [generic]
    severity: ERROR
    options:
      generic_engine: aliengrep
```

## Specifying language in rules

The `languages:` field in a rule accepts one or more language keys as a list:

```yaml
rules:
  - id: eval-usage
    pattern: eval($CODE)
    message: "Avoid eval()."
    languages: [javascript, typescript]   # applies to both
    severity: WARNING

  - id: cross-language-todo
    pattern-regex: "TODO:"
    message: "Unresolved TODO comment."
    languages: [python, java, go, ruby]   # applies to all four
    severity: INFO
```

## Specifying language on the CLI

When running inline patterns with `-e`, you must specify the language with `--lang`:

```bash
# Search Python files
semgrep -e '$X == $X' --lang=python src/

# Search JavaScript files
semgrep -e 'eval($CODE)' --lang=javascript src/

# Search with a generic pattern
semgrep -e 'SECRET_KEY = "..."' --lang=generic .
```

When scanning with `--config`, Semgrep uses the `languages:` field from each rule to determine which files to target â€” no `--lang` flag is needed.

## Language detection

Semgrep identifies a file's language using:

1. **File extension** â€” `.py` â†’ Python, `.go` â†’ Go, `.ts` â†’ TypeScript, etc.
2. **Shebang line** â€” `#!/usr/bin/env python3` selects the Python parser even for files without a `.py` extension.
3. **`--lang` flag** â€” overrides automatic detection when running inline patterns.

<Tip>
To scan a file with an uncommon or missing extension, use the `--lang` flag to explicitly specify the parser:

```bash

```
</Tip>
